qutebrowser (fork)
==================

a vim-like browser built on https://github.com/python/cpython[Python] and https://github.com/qt/qt5[Qt].

this is a personal fork. upstream lives at
https://github.com/qutebrowser/qutebrowser[qutebrowser/qutebrowser].

fork changes
------------

lazy session restore::
  tabs from a saved session load on focus instead of all at once.
  keeps startup fast when you have dozens of tabs.

tab close by string match::
  `:tab-close` accepts a string to match against tab titles/urls.

desktop file and justfile::
  `qutebrowser.desktop` for freedesktop integration.
  `justfile` for setup/run shortcuts (`just setup`, `just run`).

ephemeral sandboxed mode::
  `scripts/ephemeral` launches qutebrowser inside a bubblewrap sandbox.
  browser state lives in a tmpdir destroyed on exit. optional network
  isolation via `--isolate-net`.


example config.py
-----------------

the config file lives at `~/.config/qutebrowser/config.py`.
qutebrowser injects `config` and `c` into the namespace at runtime.

....
# pyright: reportUndefinedVariable=false
# flake8: noqa: F821

# load settings from :set commands (autoconfig.yml)
config.load_autoconfig()

# persist session across restarts
c.auto_save.session = True

# only load tabs when focused, not all at once
c.session.lazy_restore = True

# aliases
c.aliases["tc"] = "tab-close"

# spoof chrome UA on google login so they stop blocking us
config.set(
    "content.headers.user_agent",
    "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 "
    "(KHTML, like Gecko) Chrome/125.0.0.0 Safari/537.36",
    "https://accounts.google.com/*",
)

# --- security/privacy hardening ---

# block page loads on TLS certificate errors instead of prompting
c.content.tls.certificate_errors = "block"

# reject third-party cookies
c.content.cookies.accept = "no-3rdparty"

# prevent WebRTC from leaking local IPs
c.content.webrtc_ip_handling_policy = "default-public-interface-only"

# use both brave adblock and hosts-based blocking
c.content.blocking.method = "both"

# disable canvas fingerprinting by default
# toggle per-site with :set -u <pattern> content.canvas_reading true
c.content.canvas_reading = False
....
